Recipe App

A Case Study in Full-Stack Python Development

Project Pitch:

This project was a comprehensive journey through the Python ecosystem, beginning as a simple command-line application and evolving into a full-stack, deployed web application. The goal was to build a secure platform for users to create, view, and manage their own recipes. I was responsible for the entire development lifecycle, from initial data modeling to final deployment. This case study demonstrates my expertise in Python, the Django framework, and relational database management.

The Process & Challenges

The project was split into two main achievements: first, building a robust command-line tool with a proper database, and second, migrating that logic to a full-stack web framework (Django) and deploying it.

Challenge 1: Data Persistence: Evolving from Files to a Relational DB

Problem: The project started as a command-line app (Achievement 1) that needed to persist data. The initial version used Python's pickle module, which is insecure and not scalable.

Solution: I first migrated the app to use a MySQL database, interfacing with it using the SQLAlchemy ORM. This involved defining models and writing parameterized queries to prevent SQL injection. This step was critical in learning to build a robust, data-driven application and understanding the role of an ORM.

Challenge 2: Architecture: Migrating to a Full-Stack Django App

Problem: The command-line app needed to be accessible on the web. This required a complete architectural shift to a web framework.

Solution: I rebuilt the application from the ground up using the Django framework. This involved defining Django models for recipes and ingredients (including a through table for the many-to-many relationship), creating class-based views to handle logic, and building forms with query filtering. This migration created a true, scalable full-stack application.

Visual Proof: Django Class-Based Views

Challenge 3: Security & Deployment: User Authentication and Production

Problem: The app needed to be secure, ensuring users could only see and edit their own recipes. It also needed to be deployed live to the web.

Solution: I implemented Django's built-in authentication system to create secure user registration, login, and auth-protected pages. For deployment, I configured the app for Heroku, using Gunicorn as the WSGI server, WhiteNoise for serving static files, and PostgreSQL as the production database, ensuring a secure configuration with environment variables.

Visual Proof: User Authentication

The Final Solution

The final product is a secure, deployed, full-stack Django application. Users can register, log in, and perform full CRUD (Create, Read, Update, Delete) operations on their personal recipe collection. This project demonstrates a complete understanding of the Python full-stack development process, from local script to deployed web service.

Key Learnings & Future Steps
